# Security Policy

## Supported Scope

Security issues are relevant for:

- API authentication and authorization
- Document upload and storage pipeline
- Vector store and database access controls
- Prompt-injection and data exfiltration defenses
- Secrets and deployment configuration

## How to Report a Vulnerability

Do not open public issues for security vulnerabilities.

Share reports privately with maintainers and include:

1. Affected component and environment
2. Reproduction steps
3. Impact assessment
4. Suggested remediation (if available)

## Response Expectations

Target process:

1. Initial acknowledgement within 2 business days
2. Triage and severity classification
3. Mitigation plan and timeline
4. Fix validation and release communication
