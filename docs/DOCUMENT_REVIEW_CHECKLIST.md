# Document Review Checklist

Use this checklist before marking a document as `Review` or `Approved`.

## Structure

- [ ] Metadata is present (`Owner`, `Status`, `Last Updated`, `Related Docs`).
- [ ] Required sections from `DOCUMENTATION_STANDARDS.md` are present.
- [ ] Headings are specific and easy to scan.

## Technical Quality

- [ ] Behavior and design details are technically accurate.
- [ ] Assumptions and constraints are explicit.
- [ ] Failure and recovery scenarios are documented.
- [ ] Security implications are documented.
- [ ] Observability and operations requirements are documented.

## Consistency

- [ ] Terminology is consistent with existing docs.
- [ ] Cross-references use valid relative links.
- [ ] File naming follows repository conventions.

## Maintenance

- [ ] `docs/README.md` is updated if doc set changed.
- [ ] `docs/CHANGELOG.md` has an entry for meaningful changes.
- [ ] Outstanding questions have owners or next actions.
