# Document Template

Use this template for new technical documentation.

## Metadata

- `Owner`: `<team or person>`
- `Status`: `Draft | Review | Approved | Deprecated`
- `Last Updated`: `YYYY-MM-DD`
- `Related Docs`: `<relative links>`

## 1. Purpose

Describe what this document covers and why it exists.

## 2. Scope

Describe what is in scope and out of scope.

## 3. Assumptions and Constraints

List assumptions, prerequisites, and known constraints.

## 4. Design or Procedure

Describe the architecture, flow, or implementation process.

Include concrete examples such as:

- API request and response payloads
- Data model snippets
- Operational commands

## 5. Failure Modes and Recovery

List likely failures and the recovery approach for each one.

## 6. Security Considerations

Document authentication, authorization, data sensitivity, and abuse controls.

## 7. Observability and Operations

Document logs, metrics, alerts, and runbook expectations.

## 8. Open Questions

List unresolved decisions and decision owners.

## 9. Change Notes

Summarize notable updates made to this document.
